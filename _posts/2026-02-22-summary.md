### [Andrej Karpathy talks about "Claws"](https://simonwillison.net/2026/Feb/21/claws/#atom-everything) ⭐️ 8.0/10

AI researcher Andrej Karpathy identified 'Claws' as a new orchestration layer above LLM agents, highlighting concepts like persistence, scheduling, and tool calls, and specifically mentioned lightweight implementations like NanoClaw, which has a core engine of about 4000 lines of code and runs tasks in containers by default. This matters because Karpathy's endorsement and definition help formalize an emerging architectural pattern in AI, potentially accelerating development of more capable, persistent, and secure personal AI assistants that can run on local hardware, moving beyond simple chat interfaces to scheduled, multi-step task execution. A key detail is that while Karpathy praises the concept, he expresses caution about running OpenClaw specifically, and notes the proliferation of many smaller, lightweight variants like NanoClaw, Zeroclaw, and Picoclaw, which aim to be more manageable and auditable.

*Source: rss/Simon Willison*

**Background**: LLM agents are systems that use large language models to perform tasks by breaking them down, using tools (like web search or code execution), and iterating. OpenClaw, originally called Clawdbot, is a popular open-source framework released in November 2025 that acts as a gateway, connecting messaging apps to AI agents to execute tasks on a user's computer. The term 'Claw' is now becoming a category name for similar systems that emphasize personal hardware, messaging protocols, and scheduled task execution.

### [Taalas serves Llama 3.1 8B at 17,000 tokens/second](https://simonwillison.net/2026/Feb/20/taalas/#atom-everything) ⭐️ 8.0/10

Taalas, a Canadian hardware startup, announced its first product: custom silicon that runs Meta's Llama 3.1 8B model at 17,000 tokens per second, which is an unprecedented inference speed for this model. This performance is achieved through aggressive quantization, combining 3-bit and 6-bit parameters. This represents a major breakthrough in efficient AI deployment, as it dramatically lowers the cost and latency of running large language models, making them more practical for real-time applications and edge computing. It signals a shift toward specialized hardware that could democratize access to powerful AI by reducing infrastructure requirements. The implementation uses an aggressive quantization scheme that mixes 3-bit and 6-bit parameters, which is more extreme than the more common 4-bit or 8-bit quantization techniques. The company notes that its next-generation hardware will use 4-bit quantization, indicating they have a long hardware development cycle for baking out new models.

*Source: rss/Simon Willison*

**Background**: Quantization is a technique that reduces the precision of a model's numerical parameters, typically from 32-bit or 16-bit floating-point values to lower-bit formats like 8-bit or 4-bit integers, which shrinks model size and computational requirements for faster inference. Llama 3.1 8B is a popular, open-source large language model released by Meta in July 2024, based on a decoder-only transformer architecture. Inference speed, measured in tokens per second, is a critical metric for determining how quickly a model can generate text, directly impacting user experience and operational costs.

The guide emphasizes using specific, forceful language in prompts (like "deeply" and "in great detail") to prevent the AI from skimming code, and it positions this approach as a more reliable alternative to fully autonomous "unsupervised agent" workflows, which require more complex validation tooling.

Prompt engineering is the practice of designing inputs to guide large language models (LLMs) to produce better outputs, and a multi-pass strategy involves breaking a complex task into sequential steps rather than requesting a complete solution in a single prompt.

### [Quoting Thariq Shihipar](https://simonwillison.net/2026/Feb/20/thariq-shihipar/#atom-everything) ⭐️ 7.0/10

An Anthropic engineer, Thariq Shihipar, revealed that prompt caching is a foundational technology enabling the economic viability of long-running AI agent products like Claude Code, and that the team monitors cache hit rates so closely that low rates trigger SEV (Severity) alerts. The entire Claude Code system harness is built around this caching mechanism to reuse computation from previous interactions. This insight matters because it shows that advanced AI agent features are not just about model capabilities but also about production engineering that makes them cost-effective and fast enough for real-world use. The direct link between cache performance and business metrics like subscription rate limits reveals how core technical optimizations directly impact product pricing and user experience. A high prompt cache hit rate significantly decreases both latency and cost for AI inference, with industry data showing prompt caching can reduce time-to-first-token latency by up to 80% and input token costs by up to 90%. The system is sensitive to changes in the prompt, as a single character difference in the first portion of a prompt can result in a cache miss, forcing a full recomputation.

*Source: rss/Simon Willison*

**Background**: Prompt caching is an optimization technique for Large Language Models (LLMs) where the system stores and reuses the computational results of common prompt prefixes, avoiding the need to process identical inputs repeatedly. Claude Code is an AI agent product from Anthropic designed to assist with programming tasks, which involves long, multi-step interactions. SEV alerts are part of an incident severity level system used in production engineering to prioritize and respond to system issues, where a low SEV level like SEV 5 might indicate a minor, non-emergency problem.

### [ggml.ai joins Hugging Face to ensure the long-term progress of Local AI](https://simonwillison.net/2026/Feb/20/ggmlai-joins-hugging-face/#atom-everything) ⭐️ 8.0/10

The organization ggml.ai, founded by Georgi Gerganov and the entity behind the llama.cpp project, has officially joined Hugging Face to ensure the long-term progress of local AI development. This consolidation is significant because it brings the leading library for efficient, local LLM inference (llama.cpp) under the stewardship of Hugging Face, which already maintains the dominant Transformers library, potentially streamlining the entire open-source AI model ecosystem and accelerating the adoption of local AI. The joint announcement outlines specific objectives, including achieving seamless 'single-click' integration between the Transformers library and the ggml ecosystem and improving the packaging and user experience of ggml-based software to make local inference a more competitive alternative to cloud services.

*Source: rss/Simon Willison*

**Background**: Llama.cpp is an open-source project written in C/C++ that enables running large language models (LLMs) locally on consumer hardware, such as MacBooks, by using techniques like 4-bit quantization to drastically reduce model size and resource requirements. It was created by Georgi Gerganov in March 2023 and was pivotal in kickstarting the local AI movement by making models accessible without requiring specialized NVIDIA CUDA hardware. Hugging Face is a central hub for the AI community, best known for its Transformers library, which provides the standard framework for using and sharing thousands of pre-trained models.
